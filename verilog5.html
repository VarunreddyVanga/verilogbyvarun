Assignments

Continous Assignment:

  • Assign value to nets(vector or scalar)
  • They are triggered whenever the simulation causes the value of the right value of the right hand side to change
  • Keyword: "assign"

    • Eg: assign out = in1 & in2;

Procedural Assignment:

  • ‚ùïBlocking
  • ‚ùïNon-Blocking
  • Drive values onto register(Vectors and Scalar)
  • They occur within procedures such as always and initial
  • They are triggered when flow of execution reaches them (like in C)
  • All statements inside an intial statement constitue an initial block
  • An initial Block starts at time 0, executes exactly once during a simulation, and then does not exactly one one during a simulation, and then does not execute again.
  • If there are multiple initial blocks starts to execute concurrently at time 0. Each block finishes execution independently of other blocks.
  • All the behavioural statement inside an always statement constitute an always block.
  • The always statement starts at time 0 and execute the statement in the always block continously in a looping fashion if the sensitivity list is satisfied.

Ex:

  • intial begin;
  • sum = 0;
  • carry = 0;
  • end
  • always @(A or B)
  • begin;
  • Sum = A ^ B;
  • Carry = A & B;
  • end

Procedural Assignment:

  • Blocking are executed in order they are listed"
  • The assignment operator "=" is used
  • Ex:
    • reg a, b, c, reg1, reg2;
    • initial begin
    • a = 1;
    • b = 0;
    • c = 1;
    • #10 reg1 = 1'b0;
    • #5 c = reg1;
    • #5 reg2 = b;
    • end

Non - blocking Assignment

  • Contains '<= assignment operator.
  • The non blocking does not block the execution of other statements.
  • Ex:
    • always @(posedge clk)
    • begin
    • B <= A;
    • C <= B;
    • D <= C;
    • end

Types of Verilog Modelling:

  • Behavioural:
    • Procedural code, Similar to C
    • Little Structural Detail
  • Data Flow:
    • Specifies transfer to data between register.
    • Some structural information available(RTL)
  • Structural(Gate, Switch):
    • Interconnection of simple connection.
    • Purely Structural
    • More Optimized

Behavioral Modelling(Half Adder):

  • module adder_b(in1, in2, sum, carry);
  • input in1, in2;
  • output sum,carry;
  • assign{carry,sum} = in1 + in2;
  • endmodule

Data flow Design(Half Adder):

  • module adder_b(in1, in2, sum, carry);
  • input in1, in2;
  • output sum,carry;
  • assign sum = in1 ^ in2;
  • assign carry = in1 & in2;
  • endmodule

Strctural Design(Half Adder):

  • module adder_b(in1, in2, sum, carry);
  • input in1, in2;
  • output sum,carry;
  • and G0(carry, in1, in2);
  • and G1(sum, in1, in2);
  • endmodule
